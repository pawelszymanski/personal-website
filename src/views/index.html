beliefs

Beliefs

I believe in simple, self-documented and semantic
code that implements a great UX and UI design.

Principles
  • design: it is the design that makes products great, code is just a tool to achieve it
  • simplicity: know your craft, don't overcomplicate your solution thou
  • semantic, self-documented: code will be read hundreds of times, names have to be meaningful
Design
makes you develop better products faster
  • the trend is obvious: designers to programmers ratio changes
  • testing and fixing the design is always faster and less expensive than fixing code
Simple code
saves money (and headaches)
  • the less code you write the less bugs you make
  • the simpler code you write on the easier is te read it hundreds of times
  • less code means it's easier to make changes and pivot the project
  • low complexity in codebase lets new developers be productive sooner
  • using concise languages and frameworks makes software development faster
